# UGS DOMAIN KNOWLEDGE

## What UGS Is
Industrial data integration middleware: collects process data from multiple heterogeneous plant automation systems, normalizes it, stores it centrally, and redistributes it to higher-level consumers. Sits at the OT/IT boundary. NOT a SCADA, visualization tool, or process controller.

## Core Capabilities (all UGS deployments)
- Centralized data acquisition from multiple production units via configurable polling
- Multi-protocol: OPC UA (primary when confirmed), Modbus, OPC DA, TCP/IP, UDP, MQTT, DBlink/SQL
- Browser-based web configuration UI — no custom programming for standard setup
- On-premises SQL database with configurable FIFO retention and automated housekeeping
- OPC UA server role exposing acquired data to downstream consumers
- Role-based user accounts, full user action audit trail
- Email alerting/reporting (value-added, project-dependent)
- Remote support post-commissioning (value-added, duration project-dependent)
- On-site training at commissioning (duration and count project-dependent)

## Business Drivers
Customers implement UGS to: eliminate siloed data per unit, replace point-to-point integrations, enable plant-wide KPI visibility, create digitalization foundation (MES, analytics, AI/ML, cloud), reduce manual reporting, apply consistent data governance.
UGS never guarantees production performance, product quality, or yield.

## Plant Environment
Continuous-process manufacturing, multiple production units with existing PLCs, HMIs, and data acquisition servers (make and presence are project-dependent — do not assume). OT network (L1 LAN, L2 LAN) separated from IT. OT firewalls always buyer scope.
UGS integration is strictly read-only. It cannot configure existing L1/L2 systems — buyer or buyer's OEM must do that.
