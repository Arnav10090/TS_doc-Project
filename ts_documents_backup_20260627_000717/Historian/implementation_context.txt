Implementation phases: M1 — Design & Tag Engineering (~4w): finalize tag lists, collection frequencies, deadbands, storage sizing; M2 — Hardware/Software Delivery (~6-8w): server delivery and software licensing; M3 — Staging & FAT (~2w): Historian installation and test with simulated tags; M4 — Installation & Tag Configuration (~6w): deployment on site, connection to live L1/L2 sources, configuring tags; M5 — Commissioning & Handover (~2w): validating data collection over a period (e.g., 7 days), client tools training, handover. Total durations indicative.

FAT: Verification of historian installation, database creation, and ability to collect simulated data via OPC.

Buyer obligations: Provide finalized I/O tag list with polling frequencies and engineering units before M1 completion; provide OPC Server access or PLC communication details; ensure network connectivity between L1/L2 sources and the Historian server; allocate necessary IP addresses.

Value addition: Development of 3-5 standard process dashboards/trends for key plant metrics.

Standard exclusions: L1 PLC programming to expose new tags; L2 SCADA modifications; enterprise ERP integration (unless explicitly scoped); custom reporting application development.

Risks: Changes to the tag list during M4 can cause significant delays. Poor network quality between L1 and Historian can result in data loss or gaps.
