Architecture Options:
Option A (Direct): UGS server polls each unit's data acquisition server centrally. No edge hardware. Use when network reliability is high.
Option B (HX per unit): HX edge controller at each unit feeds central UGS server. Use when network reliability cannot be guaranteed.
Layers (both options): Field → L1 (PLCs, PDA servers, L1 LAN) → L2 (supervisory, L2 LAN) → OT Boundary (firewalls, buyer scope) → IT (UGS server, managed switch, downstream consumers) → Remote Access (buyer-controlled VPN).

Protocols: Primary: OPC UA. Also: OPC DA, Modbus RTU/TCP, TCP/IP raw, UDP, MQTT, DBlink/SQL. Protocol selection is project-dependent. Outbound: UGS OPC UA server for live subscriptions; SQL for batch (heavy queries degrade acquisition performance — discourage).

Data Storage: Centralized on-premises SQL database. Retention: configurable FIFO ([RETENTION_PERIOD] from metadata). Automated housekeeping prevents manual DBA effort. No write-back to source systems.

Hardware: enterprise tower/rack server (sizing by tag count, polling frequency, retention — never state specific RAM/storage), FHD monitor, HIL-approved managed switch, network cabling, HX controller + junction box (if Option B).
