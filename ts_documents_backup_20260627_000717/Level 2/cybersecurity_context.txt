Security architecture: Defense-in-depth approach with network segmentation (L1/L2/L3 zones), firewall-enforced access controls, demilitarized zone (DMZ) deployment for L2 servers, VLAN isolation, intrusion detection systems (IDS) optional, encrypted data transmission (TLS 1.2+), secure authentication mechanisms (Active Directory, LDAP, local accounts with password policies).

Application security: Role-based access control (RBAC) with operator/supervisor/engineer/administrator roles; audit logging of all user actions and data modifications; session timeout and automatic lockout; encrypted database connections; secure API endpoints with token-based authentication; input validation and SQL injection protection; secure credential storage (hashed passwords, encrypted connection strings).

Patch management: System updates and security patches are the responsibility of the facility IT/OT team unless explicitly covered under managed services contract. Vendor provides tested patch recommendations quarterly; emergency security updates issued as needed; patching procedures documented to minimize production impact; testing in non-production environment recommended before production deployment.

Remote access: Vendor remote support requires secure VPN connection provisioned by client; multi-factor authentication (MFA) recommended; all remote sessions logged with timestamp and user identification; remote access limited to L2 application layer only — no direct L1 control system access; signed non-disclosure agreements (NDA) and cybersecurity protocols enforced.
