## Implementation Phases

Phase 1 — Engineering Survey: SELLER and BUYER jointly study existing systems, gather basic data (yard layouts, crane details, interface specifications, system inventory). Buyer provides site access, technical team, and existing system documentation.

Phase 2 — Engineering Review Meeting: Alignment on scope, interface requirements, yard configuration, and project schedule. Kick-off of Basic Engineering.

Phase 3 — Design and Build (Basic Engineering + Detailed Design + Software Development): System design document produced and submitted for BUYER approval. After BUYER approval, software development begins. OBSERVED duration: [SW_DEV_DURATION] from design document approval. Delays in design approval cause proportional project delay.

Phase 4 — Pre-Commissioning: Software and hardware readiness verification before site deployment. Internal testing and quality assurance.

Phase 5 — Commissioning and Launch: SELLER supervisors deployed to site. Hardware installation (by BUYER), software deployment, integration testing with live Level-2 and MES/ERP systems, crane integration testing, operator training, snag resolution, and handover. Requires crane availability and site readiness from BUYER.

Phase 6 — Handover: Submission of documentation package, Work Completion Certificate issuance, remote support period begins.

Critical path gating points:
- M1 (Design Approval): BUYER must approve System Design Document; this gates software development start.
- M2 (Site Readiness): Crane hardware, network infrastructure, and master data must be ready before on-site commissioning begins.
- M3 (Work Completion Certificate): Marks formal project completion; gates start of remote support period.

## Testing

Pre-Commissioning / FAT (if applicable): Functional testing of software flows and integrations in SELLER environment or staging environment against simulated inputs before deployment to site. Verifies configured yard rules, interface message handling, visualization, and reporting functions.

Site Acceptance / Commissioning Testing: Live integration testing at site with actual Level-2, MES/ERP, Weigher, and crane systems. Tests include: material tracking end-to-end, crane instruction display and feedback, yard visualization accuracy, report generation, user access control, and interface message verification. BUYER assigns crane operators, weighbridge operators, yard supervisors, and QC personnel for integration testing.

Acceptance artifacts: All documentation per section 3.3 scope (Screen Design Document, Hardware Specifications, Software Specifications, Operation Manual); Work Completion Certificate issued by BUYER.

## Risks

1. System Design Document approval delay — gates software development; BUYER-owned risk; mitigate by setting approval turnaround deadline in TS.
2. Crane hardware or sensor procurement delay — BUYER scope items; gates commissioning; define as formal prerequisite.
3. Level-2 or MES/ERP interface unavailability at commissioning — BUYER must make OEM engineers available; define as buyer obligation.
4. Network infrastructure not ready at commissioning — Wi-Fi coverage, VLAN, IP assignments, firewall rules, port openings must be ready; BUYER obligation.
5. Master data not available — yard locations, material masters, product codes, QC codes, customer masters must be provided complete and accurate by BUYER before commissioning.
6. Crane availability constraint — commissioning requires cranes available in working condition; crane downtime for maintenance or production priority can delay commissioning.
7. Integration interface parameter changes — L2 or ERP schema/API changes discovered late require change order; mitigate by freezing interface specs during Basic Engineering.
8. Plate identification method not finalized — barcode/RFID/OCR selection deferred to detailed engineering; delay in finalization affects software development.
9. Scope creep from undiscovered cranes or yard areas — define scope by named yards and crane count; additions require change order.
10. Remote access link not established — required for remote support; BUYER obligation; NDA required before sessions.

## Standard Assumptions (CORE / COMMON)

- Level-2 systems can expose required data interfaces for PMYMS integration; upgrades to L2 systems for this purpose are BUYER scope.
- All integration parameters (interface schemas, API endpoints, communication protocols) finalized during detailed engineering.
- BUYER provides complete and accurate master data before commissioning.
- Yard scope is defined by the named yards, zones, and crane count agreed at the time of TS signing; additions are a change order.
- IP addresses, hostname assignments, VLAN/network routing provided by BUYER during engineering.
- Crane hardware (position sensors, lock/unlock sensors, zeroing sensors, wireless APs, HMIs) procured and installed by BUYER within agreed schedule.
- All crane systems are in working condition before commissioning activities begin.
- Site network readiness (Wi-Fi coverage, firewall rules, port openings) is BUYER responsibility and must be completed before on-site commissioning.

## Standard Exclusions (CORE)

- Interfaces with external systems or devices not explicitly named in the agreed and signed TS.
- Operating system patches, antivirus updates, and software maintenance after project delivery (post-delivery patch management is BUYER scope).
- Source code of SELLER's core technology platform.
- Modifications to existing Level-1, Level-2, MES, ERP, or FCE systems (these are BUYER scope).
- Software or hardware modifications to existing crane PLC, drive systems, or crane control infrastructure.
- Erection, civil, or mechanical works of any kind.
- Hardware and software not listed in the agreed scope of supply.
- Mechanical equipment supply, modification, or maintenance.
- Troubleshooting support for mechanical, process, or operational issues outside the Solution scope.
- L2 and PDA software and related hardware.
- Firewall, managed switches, and other networking infrastructure.
- Performance guarantees on productivity, yield, quality, or process capability.
- Supervision or management of BUYER personnel, contractors, or agents.
- Any warranty, guarantee, or liability regarding production output, product quality, or yield.
