Role: Responsibility matrix showing who (BUYER, SELLER, or joint) owns each activity across project phases for all scope items.
Required: Architecture option (affects HX Controller and crane interface line items); confirmed integration list; confirmed hardware scope items.
Infer: Standard responsibility pattern — SELLER: Basic Engineering, Detail Design, Supply, and Commissioning for all SELLER-scope hardware and software items; BUYER: Basic Data, erection of hardware at site, firewall and network configuration, L2 and ERP system-side interface modifications; Joint (B/S): project execution coordination, overall system design. For crane subsystem: BUYER supplies hardware (sensors, APs, HMIs, Gateway hardware); SELLER designs, engineers, and commissions. For interfaces: L2 side = BUYER; PMYMS side = SELLER.
Never invent: Responsibility assignments without project-specific basis.
Structure: Table with columns: No., Item, BD, BE, DD, SU, ER, COM. Codes: B = Buyer, S = Seller, B/S = Joint. Categories: (1) Services (project execution, system design, testing), (2) System Engineering (documentation), (3) Hardware (PMYMS servers, client PCs, GSM Modem, HX Controller), (4) Software (all software components), (5) Training, (6) Pre-Engineering Activities, (7) Documents, (8) Crane System (sensor and hardware items with B for supply, S for engineering), (9) Interface with Other Systems (each interface appears twice: source-system side = B, PMYMS side = S). Network cables and accessories = B for all phases. Android mobile devices = B for all phases.
