## Architecture Options

Option A — Centralized Server with Direct Integration:
PMYMS application server (online + backup pair) connects directly to Level-2 systems, MES/ERP, and Weigher systems via API/database interfaces over the plant network. Crane position and lock/unlock signals received via crane-side gateway hardware and wireless network. Suitable when network infrastructure is reliable and plant systems are accessible from server location.

Option B — With HX Controller (Unified Gateway) for Crane Integration (COMMON):
SELLER-supplied HX Controller (proprietary unified gateway) installed to bridge crane subsystem signals (position sensors, lock/unlock sensors) to the PMYMS server. Provides protocol normalization and local signal buffering. Used when direct integration from server to crane instrumentation is impractical.

Both options use:
- PMYMS Application Server (online + backup) for core software
- Client workstations for operator and supervisor access
- Web browser interface (no client software installation required beyond standard browser)
- Android mobile devices for field access (buyer scope)
- Network infrastructure managed by buyer (Wi-Fi or wired; finalized during detailed engineering)

## Data Acquisition

Material and production data flow into the Solution from:
- Plate Mill Level-2: Plate production data (rolling events, plate dimensions, grades)
- Furnace Level-2: Slab tracking data (charging, discharge, heat number)
- Plate Leveller Level-2: HTC and Non-HTC production completion data
- MES/ERP: Order data, shipping plans, quality status, customer master data
- Crane subsystem: Position coordinates (X, Y axes), lock/unlock signals (tong/magnet feedback)
- Weigher system: Weight data for dispatch confirmation

Outbound data from Solution to external systems:
- MES/ERP: Movement confirmations, stock updates, loading memo confirmations, shipment records
- Optional: SMS gateway for alert notifications

All inbound and outbound messages (telegrams) are logged internally for troubleshooting.

## Data Storage

Centralized on-premises relational database (MS SQL Server or alternative; selection at SELLER discretion) on the PMYMS server infrastructure. Data includes:
- Real-time and historical yard inventory positions (zone, row, pile, layer)
- Material attributes and production traceability records
- Crane instruction and execution logs
- Interface message logs (all inbound/outbound telegrams)
- Event history for every material action
- User action logs (manual overrides, exceptions)
- Reports and dashboard data

No write-back to Level-1 or Level-2 source systems from PMYMS. PMYMS consumes L1/L2 data; it does not modify it.

Backup server provides redundancy. Backup software (OBSERVED: Acronis or equivalent) manages server-level backup. Retention and housekeeping policy are project-dependent.

## Integration Patterns

1. Plate Mill L2 → PMYMS: L2 side integration modifications are BUYER scope; PMYMS side is SELLER scope.
2. Furnace L2 → PMYMS: Same pattern — L2 side is BUYER scope; PMYMS side is SELLER scope.
3. MES/ERP (L3/SAP) ↔ PMYMS: L3/SAP side modifications are BUYER scope; PMYMS side is SELLER scope.
4. Crane subsystem → PMYMS: Crane-side hardware (sensors, wireless APs, HMIs, Gateway hardware) is BUYER supply scope; SELLER provides engineering, commissioning, and gateway integration.
5. Weigher system → PMYMS: Integration details finalized in detailed engineering.

Any data required from Level-1 systems must be routed through the existing Level-1 to Level-2 integration chain before reaching PMYMS. PMYMS does not connect directly to Level-1 PLCs.

Firewall configuration and network security remain exclusively BUYER scope.
