Gantt Phases (all durations indicative only — not commitments):
Initiation → Pre-Engineering (M1: buyer tag list delivery) → Basic Engineering (M2: buyer design approval) → Detailed Design → FAT (M3: FAT sign-off) → Shipping → Site Preparation (M4: site ready) → Commissioning → Handover (M5: work completion cert).

Scale examples (observed, not defaults): small project (few units, no HX) ≈10–14 weeks; medium (several units, HX) ≈14–20 weeks; large (many units, complex) ≈18–26 weeks.

Draw.io format: week-based JSON with fields: task name, phase, start_week (integer, week 1 = PO acceptance), duration_weeks (integer), milestone (bool), dependencies (array of predecessor indices). Backend converts to mxGraph XML.

Critical Path: M1 (tag list delivery) and M2 (design approval) are buyer-gated milestones. Buyer prerequisite delays cause proportional delivery delays — state this explicitly.

Shutdown Gantt: UGS commissioning is typically non-intrusive — no full shutdown required. Brief panel/cable access managed as permits-to-work. If metadata confirms a shutdown is required, generate week-based tasks for shutdown-window activities only. If no shutdown required, state explicitly in one sentence.

Mandatory language: All durations are indicative. Final schedule agreed at PO acceptance. Buyer prerequisite delays (M1, M2, M4) will proportionally extend delivery.
