## division_of_eng
Role: Responsibility matrix showing who owns each project activity across phases (Basic Data, Basic Engineering, Detail Design, Supply, Erection, Commissioning).
Required: Architecture option (affects hardware line items); confirmed production units; confirmed tool scope.
Infer: Seller provides: OT cybersecurity assessment activities, architecture design, tool supply, IDS/SRA/VM tool configuration and commissioning, policy/SOP documentation, training delivery, onsite SV monitoring. Buyer provides: plant coordination, site access, network infrastructure (cables, switches, server room, firewall rules, SPAN configuration), existing system documentation, plant and process domain subject matter expertise, project management, all approvals and stakeholder availability. Shared (S/B): scope definition, asset discovery activities, network architecture validation, acceptance and sign-off.
Never invent: Responsibility assignments without project basis.
Structure: Table with columns: No., Category, Item, BD, BE, DD, SU, ER, COM. Codes: B=Buyer, S=Seller, S/B=Joint (primary S or B with support from the other). Categories: Supply (IDS hardware, SRA hardware, software licenses), Services (assessment, architecture, configuration, training, SV support), Documents (policy/SOP, project plan, asset register, reports), Infrastructure (network, power, rack — Buyer scope), Management (project coordination — primarily Buyer with Seller support).
