Level 2 Production Management System (L2 PMS) sits between Level-1 plant-floor control (PLC/HMI) and Level-3 enterprise systems (ERP/MES/business systems). It orchestrates manufacturing execution for one or more production lines: receives production-unit specifications, converts them into machine setpoints, tracks the unit of production through the line, captures actual results, and exchanges this data with L1 and L3. NOT a SCADA or visualization-only tool, NOT the L1 control system itself, and NOT a data-acquisition/historian middleware platform — that is a distinct product category that may sit alongside or beneath an L2 PMS but serves bulk tag collection and redistribution rather than production-order orchestration.

Core capabilities (all L2 PMS deployments): Primary Data Input (PDI) — receives production-unit specification data from L3 or manual HMI entry, validates against limits and congruency, alarms on bad data, supports sequence changes and manual override; setup/schedule function — converts PDI into machine reference setpoints using buyer-provided recipe-based look-up tables/algorithms (speed, tension, reduction, force, power references for L1); adaptive/learning function (where the process supports it) — compares calculated setpoints against actual results, derives compensation coefficients, updates the schedule over time (scope project-dependent, may be limited to a single parameter); Production Data Output (PDO) / delivery file control — captures final results at unload/completion and sends to L3; equipment/tooling management (roll, tool, die data tracking, cumulative usage, event-driven transitions); downtime management (line stoppage classification); reporting function (production and performance reports); data link with L1 PLC (read/write setpoints and status) and data link with L3 computer (PDI in, PDO out).

Optional/value-added capabilities (project-dependent): mobile/HHT application for executives/supervisors; data visualization and analytics dashboards (customized during engineering); SMS alert system; email alert and reporting system; remote post-commissioning support (duration project-dependent); on-site customer training (duration and count project-dependent).

Business drivers: move from manual/paper or siloed production tracking to fact-based decision making, give line managers live and historical visibility into process performance, fine-tune process setpoints beyond static recipe tables, establish traceability for each production unit, lay groundwork for further digitalization (dashboards, AI/analytics, scheduling tools). An L2 PMS never guarantees productivity, quality, or yield outcomes.

Plant environment: discrete or semi-continuous manufacturing with one or more production lines/units, each with its own existing L1 system (PLC/HMI; make and model are project-dependent — do not assume). An existing L3/ERP system may or may not be present; if absent, the L2 PMS provides manual PDI entry and production-sequence entry as a fallback. OT network separated from IT; firewalls are buyer scope.
